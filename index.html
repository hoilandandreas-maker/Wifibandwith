Where do I find the dBm value?

Browsers cannot read WiFi signal strength directly, so it is entered by hand. On Android: Settings → Network & internet → Internet → tap the connected network — signal strength is shown there (some phones show dBm, others show it under “Advanced”). A free WiFi-analyzer app also shows live dBm. On iPhone there is no built-in readout; leave the field empty or use the Airport Utility scanner. Rule of thumb: −50 excellent · −60 good · −70 fair · −80 weak.

Help

Install on the phone (works offline)

Open this page in Chrome (Android) or Safari (iPhone) and choose Add to Home Screen. After the first visit the app itself loads without any internet connection; only the speed test needs a reachable server.

Measuring pure WiFi throughput (no satellite link)

The default servers are on the internet, so offshore the result is limited by the shore/VSAT link. To measure only the WiFi link, run a test server on a laptop on the same network:

dd if=/dev/urandom of=test.bin bs=1M count=200
npx http-server -p 8080 --cors

then set the custom download URL to http://<laptop-ip>:8080/test.bin. Note: the HTTPS-hosted app cannot call plain http:// LAN addresses (browser mixed-content rule). For LAN tests, serve this app from the same laptop instead — it is a plain static site; see the README in the repository.

What this app can and cannot measure

Browsers expose no API for WiFi signal strength (RSSI), channel or BSSID — that is why dBm is entered manually. What is measured automatically: download/upload throughput, latency and jitter against the chosen server, plus the browser's own connection estimate (type, downlink, RTT) where available.
